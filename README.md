# 时间轴记事工具（本地文件版）

这是一个简单易用的时间轴记事工具，支持本地存储和数据导出导入功能，无需任何在线服务。

## 功能特点

✅ **本地存储**：数据完全保存在本地，无需任何在线服务
✅ **数据导出导入**：支持将数据导出为JSON文件，方便在不同设备间同步
✅ **操作简单**：无需编程知识，直接打开HTML文件即可使用
✅ **功能完整**：添加、修改、删除事件，按时间顺序展示
✅ **单文件使用**：所有代码都在一个HTML文件中，方便保存和分享

## 如何使用

### 第一步：打开工具

1. 找到 `timeline-local.html` 文件
2. 双击文件，或右键选择"打开方式" → 选择您常用的浏览器（如Chrome、Edge、Firefox等）
3. 浏览器会自动打开时间轴工具

### 第二步：添加事件

1. 在"事件名称"输入框中输入事件名称（必填）
2. 点击"事件时间"选择框，选择事件发生的日期和时间（必填）
3. 在"备注"输入框中填写可选的备注信息（可选）
4. 点击"添加事件"按钮
5. 事件会自动添加到时间轴上，按时间顺序排列（最新的在前面）

### 第三步：查看事件

- 添加的事件会显示在时间轴上，每个事件卡片包含：
  - 事件名称（粗体显示）
  - 格式化的日期和时间
  - 备注（如果有）
  - 修改和删除按钮

### 第四步：修改事件

1. 找到要修改的事件卡片
2. 点击卡片右下角的"修改"按钮
3. 事件卡片会进入编辑模式（背景变为浅黄色）
4. 修改事件名称、时间或备注
5. 点击"保存"按钮保存修改，或"取消"按钮放弃修改

### 第五步：删除事件

1. 找到要删除的事件卡片
2. 点击卡片右下角的"删除"按钮
3. 会弹出确认对话框，点击"确定"删除事件，"取消"则保留

### 第六步：导出数据（换设备时使用）

当您需要在另一个浏览器或设备上使用时间轴工具时，可以先导出数据：

1. 在当前浏览器中打开时间轴工具
2. 点击顶部的"导出数据"按钮
3. 浏览器会自动下载一个JSON文件（文件名格式：`timeline-events-日期.json`）
4. 将这个JSON文件保存到您的电脑或U盘上

### 第七步：导入数据（在新设备上使用）

在新浏览器或新设备上使用导出的数据：

1. 在新浏览器中打开 `timeline-local.html` 文件
2. 点击顶部的"导入数据"按钮
3. 选择您之前导出的JSON文件
4. 会弹出确认对话框，询问是否合并现有数据：
   - 点击"确定"：将导入的数据与现有数据合并（不会重复）
   - 点击"取消"：用导入的数据替换现有数据
5. 导入成功后，会显示导入的事件数量

## 数据存储说明

1. **单设备使用**：数据会自动保存在浏览器的本地存储中，关闭页面再重新打开，数据会自动加载
2. **跨设备使用**：通过导出和导入JSON文件实现数据同步
3. **数据安全**：数据完全保存在本地，不会上传到任何服务器，安全可靠
4. **数据备份**：建议定期导出数据备份，以防浏览器数据丢失

## 常见问题

### Q: 关闭浏览器后数据会丢失吗？
A: 不会。数据会自动保存在浏览器的本地存储中，关闭页面再重新打开，数据会自动加载。

### Q: 换浏览器或设备后数据会丢失吗？
A: 直接换浏览器或设备会丢失数据，但您可以通过导出和导入功能来同步数据。

### Q: 如何将数据从旧设备转移到新设备？
A: 
1. 在旧设备的浏览器中打开工具，点击"导出数据"，下载JSON文件
2. 将JSON文件复制到新设备
3. 在新设备的浏览器中打开工具，点击"导入数据"，选择下载的JSON文件

### Q: 可以在手机上使用吗？
A: 可以。直接在手机浏览器中打开HTML文件即可使用，但由于屏幕较小，操作体验可能不如电脑。

### Q: 导出的JSON文件可以用什么打开？
A: 可以用文本编辑器（如记事本、Notepad++、VS Code等）打开查看或编辑，但不建议手动编辑，以免格式错误。

### Q: 导入数据时提示"无效的数据格式"怎么办？
A: 请确保您导入的是之前通过本工具导出的JSON文件，不要手动编辑文件内容，否则可能导致格式错误。

## 技术说明

- **文件格式**：HTML文件，包含完整的HTML、CSS和JavaScript代码
- **数据格式**：JSON格式，包含版本信息、导出日期和事件数据
- **浏览器支持**：支持所有现代浏览器（Chrome、Edge、Firefox、Safari等）
- **无需安装**：直接打开HTML文件即可使用，无需安装任何软件

## 注意事项

1. **数据备份**：建议定期导出数据备份，以防浏览器数据丢失
2. **文件安全**：请妥善保管导出的JSON文件，不要分享给他人，以免泄露个人信息
3. **浏览器兼容性**：建议使用最新版本的浏览器，以获得最佳的使用体验
4. **不要手动编辑JSON文件**：手动编辑可能导致格式错误，导致无法导入
5. **同一浏览器多标签页**：在同一浏览器的多个标签页中打开工具，数据会自动同步
